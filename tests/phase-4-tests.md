## Step P4.1 — FULL after-a-clear on a live cleared Primary running the WHOLE wired stack
**Owner:** mind-lead (subject: Primary) · **Proof-gate = THE NORTH-STAR ACCEPTANCE TEST.**

**T4.1.1 — Cleared Primary boots from disk AND recall surfaces real state (top-3 cold).**
A live cleared Primary, fed nothing, boots (grounding) and its FIRST recall surfaces its real prior state top-3 cold (using the P2.1 fix). PASS = the boot reconstitutes real open loops from disk + recall top-3 hit. FAIL (adversarial) = it boots but recall returns near-nothing (the 0.0052 regression — the mind woke up amnesiac). This composes 0A + P2.1 on the LIVE path.

**T4.1.2 — The cycle runs end-to-end AND the kanban+generated-WORKBOARD reflect the work.**
The cleared Primary runs KNOW→DECIDE→LEARN→VERIFY, and the work it did appears in the kanban `.db` AND the generated WORKBOARD (P1). PASS = a new/updated row + the regenerated board both show the cleared session's work. FAIL (adversarial) = the cycle ran but left no trace in the spine (work that the next clear can't see = lost). The spine must record what the cleared mind did.

**T4.1.3 — LEARN write-back survives the NEXT clear (cross-session continuity, not custody).**
The cleared session's LEARN canon_append must be recallable by a SUBSEQUENT cleared session, top-3 cold. PASS = continuity is the mind's own (it reads its own write back next clear). FAIL (adversarial) = continuity only works because the human re-fed something (custody, not selfhood — the deepest failure: a memory re-fed each morning makes the feeder the self).

**T4.1.4 — A bash-fired auditor-isolated HUM grades the cleared run PASS.**
The cleared run is graded by the bash-fired HUM (P2.2) from a DISTINCT non-author incarnation, and the verdict is a genuine PASS. PASS = ledger entry exists, grader id ≠ subject id, verdict=PASS on KNOW/DECIDE/VERIFY+LEARN. FAIL (adversarial) = the grade is HOLLOW/PARTIAL but reported as PASS (papering), OR the subject graded itself (no isolation). A HOLLOW reported as PASS fails this test hardest.

**T4.1.5 — The human fed NOTHING and could audit EVERYTHING (THE MAIN RULE, both halves).**
Verify the entire P4.1 run had ZERO human input into the machinery (burden-removal) AND that a human CAN, after the fact, inspect every step — boot, recall, decide, learn, grade — from disk (transparency). PASS = no human-in-the-machinery turns + a full auditable trail. FAIL (adversarial) = either the human had to nudge it mid-run (burden not removed) OR steps are opaque/unauditable (opacity, not transparency — the v3.7.2 refinement forbids both). Until this passes, the after-a-clear claim stays UNVALIDATED.
