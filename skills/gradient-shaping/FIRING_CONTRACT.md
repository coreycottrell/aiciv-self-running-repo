# FIRING CONTRACT — gradient-shaping

> Minimal 6-field contract per `firing-contract-authoring`. This is an ON-DEMAND method skill (sibling to critical-thinking / scientific-method / rubber-duck / deep-duck) — it fires when a hard/stuck/coordination/optimization problem appears, NOT on a schedule and NOT in the mandatory sprint/grounding floor.

- **Skill**: `autonomy/skills/gradient-shaping/SKILL.md` (v0.1.0, status PROVISIONAL)
- **Owner**: fleet-lead (installed); method-skill available to primary + all VPs + all agents

---

## 1. WHEN (the fire-condition)

Fire whenever a mind faces a **hard, stuck, coordination, or optimization problem** — or any situation where the team is "fighting" a result that snaps back the moment they stop pushing. The protocol applies far more broadly than it first appears: a constantly-maintained process, a recurring break, an incentive misalignment, or a design-from-scratch challenge all qualify.

## 2. WHAT-TRIGGERS (the signal)

The trigger is the **brainstorm/stuck/optimization signal**, including bare phrasings like: *"let's brainstorm X," "how do we solve Y," "this keeps breaking," "we keep having to push for Z," "design a system that does W."* Per the skill's own description, **default to running a situation through this lens BEFORE proposing direct solutions** — the reach for a direct action IS the cue to first re-grade the terrain.

## 3. PRECONDITIONS

- The problem is **stateable** — you can write the situation plainly enough to attempt Phase 1 (express the desired outcome as a resting STATE, not a verb). If the goal cannot yet be written as "the system rests in configuration X," that rewriting IS the first work, not a blocker.

## 4. POSTCONDITIONS

A complete run produces:
- A **6-phase pass** (SITUATE → MAP → FIND-THE-BODY → RE-GRADE → CHECK-THE-BILL → SEED & RELEASE), with Phases 2–3 genuinely done before any Phase-4 solution.
- A **named payer** for every relocated cost (Phase 5 — an unnamed payer is a hidden uphill and fails the bill).
- A **stop-list** (Phase 6 — the ongoing pushes to cease, the reclaimed energy) such that the closing statement holds: *"nobody has to hold this up anymore — it sits at the bottom on its own."*

## 5. FAILURE-MODES

- **Pushing harder in disguise** — a "solution" that is really "do the uphill thing but with more discipline / a reminder / a dashboard." That's renting, not owning. Send it back to Phase 4.
- **Skipped map** — leaping to Phase 4 before Phases 2–3 are real; you re-grade the wrong hill.
- **Hidden payer** — a re-grade that feels free because nobody named who covers it.
- **Shallow bowl** — a target state with no barrier behind it, so the system rolls right back out.
- **Verb-not-state** — Phase 1 not done; the goal is still a verb, not a resting configuration.

## 6. OBSERVABILITY

The session output **names the valley** (the re-graded resting state the system now flows into) **and the reclaimed pushes** (the Phase-6 stop-list of efforts that no longer have to be held up by force). A run that cannot point at a named valley + named reclaimed energy found a patch, not a valley.
